# 絵文字の「文字数」はプラットフォームで違う

## きっかけ

文字を指定回数繰り返すAPIを作っていたとき、🍣の `.length` が2になることに気づいた。

```js
"a".length  // → 1
"あ".length // → 1
"🍣".length // → 2
```

なんで？と思って調べたら、JavaScriptの `.length` はUTF-16のコードユニット数を返していて、絵文字はサロゲートペア（2つのコードユニット）で表現されるから2になるらしい。

## HTMLの入力フォームも同じ

ブラウザの `<input maxlength="10">` も `.length` ベースなので、🍣は2文字扱いになる。つまり `maxlength="10"` のフォームには🍣は5個しか入らない。

## Twitter/Xは違った

試しにTwitter（X）で🍣を入力してみたら、1文字としてカウントされていた。Twitterは書記素クラスター単位でカウントしているっぽい。人間の感覚に近い。

## JavaScriptで「正しい」文字数を取る方法

```js
// UTF-16コードユニット数（.length）
"🍣".length                    // → 2

// コードポイント数
Array.from("🍣").length        // → 1
[..."🍣"].length               // → 1

// 書記素クラスター数（結合絵文字にも対応）
[...new Intl.Segmenter().segment("👨‍👩‍👧‍👦")].length  // → 1
```

`Intl.Segmenter` を使えば、👨‍👩‍👧‍👦のような結合絵文字（家族絵文字）も1文字として数えられる。

## UTF-8バイト数もまた違う

```js
new Blob(["🍣"]).size  // → 4（UTF-8で4バイト）
new Blob(["a"]).size   // → 1
new Blob(["あ"]).size  // → 3
```

DBの `VARCHAR` がバイト数制限の場合はこっちが効いてくる。

## まとめ

「文字数」と一口に言っても、どの単位で数えるかで結果が変わる。

| 方式 | "a" | "あ" | "🍣" | "👨‍👩‍👧‍👦" |
|---|---|---|---|---|
| `.length`（UTF-16） | 1 | 1 | 2 | 11 |
| `Array.from().length`（コードポイント） | 1 | 1 | 1 | 7 |
| `Intl.Segmenter`（書記素） | 1 | 1 | 1 | 1 |
| `Blob.size`（UTF-8バイト） | 1 | 3 | 4 | 25 |

入力フォームのバリデーションを実装するときは、どの「文字数」を使うのか意識しておくと事故が減りそう。
